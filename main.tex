\title{\textbf{GCN-GrabCut: Graph Convolutional Trimap Prediction\\
       for Interactive Image Segmentation}}
\author{Haniel Ulises V\'asquez Morales}
\date{\today}

\begin{document}
\maketitle

\begin{abstract}
Classical GrabCut requires a hand-drawn trimap that labels every pixel as definite foreground,
definite background, or unknown — a bottleneck that limits usability in practice.
We present \textbf{GCN-GrabCut}, a hybrid framework that replaces manual trimap annotation
with an automatically predicted one.
A superpixel Region Adjacency Graph (RAG) is constructed from the input image using SLIC,
and sparse user clicks are propagated across the graph via a deep residual Graph Convolutional
Network (GCN).
The predicted trimap is then passed directly to OpenCV's \texttt{cv2.grabCut} for pixel-level
refinement.
Our primary model, \textbf{ResGCNNet}, combines pre-norm residual GCN blocks with per-layer
EdgeGate modulation, multi-scale SAGEConv aggregation, and an attention-weighted global context
module, achieving accurate trimap prediction from as few as five foreground and five background
clicks.
This work is a research exploration of GNN-guided interactive segmentation, contributing
an open-source implementation, three model variants, and a reproducible training pipeline
on the BSDS500 benchmark.
\end{abstract}

\section{Introduction}

Interactive image segmentation occupies a practical middle ground between fully automatic
methods and manual pixel-level annotation.
GrabCut~\cite{rother2004grabcut} remains a canonical example: it asks the user to supply a
bounding box or trimap, then iteratively refines a foreground mask by fitting Gaussian Mixture
Models (GMMs) and solving a min-cut on a pixel graph.
Despite its age, GrabCut is still widely used as a post-processing step in annotation
pipelines~\cite{boykov2001fast}.

The central limitation of GrabCut is its dependence on a high-quality initialisation.
In bounding-box mode, large or asymmetric objects frequently produce unsatisfactory masks.
In trimap mode, the annotation burden is shifted entirely to the user.
Our key observation is that \emph{a handful of clicks carries enough information to predict a
high-quality trimap automatically}, provided the model can propagate these sparse signals
across the image structure.

Graph Neural Networks~\cite{kipf2017semi,wu2020comprehensive} are a natural fit for this
signal propagation task.
By treating superpixels as nodes and spatial adjacency as edges, a GCN can pass foreground
and background cues between neighbouring regions, producing a dense trimap prediction from
sparse input.

\medskip
\noindent\textbf{Contributions.}
\begin{enumerate}[label=(\roman*), itemsep=2pt]
  \item We formulate trimap prediction as node classification on a superpixel RAG, enabling
        end-to-end learning from click annotations.
  \item We propose \textbf{ResGCNNet}, a deep residual GCN with EdgeGate modulation,
        HintBooster conditioning, and attention-pooled global context.
  \item We provide two lighter variants (\textbf{GCNTrimapNet} and \textbf{GATTrimapNet})
        sharing the same inference interface, enabling architecture comparison under identical
        conditions.
  \item We release an open-source implementation with a reproducible training pipeline,
        configuration system, and visualisation utilities.
\end{enumerate}

\section{Background}

\subsection{GrabCut}

GrabCut~\cite{rother2004grabcut} minimises an energy function over a Markov Random Field
(MRF) defined on the pixel lattice:
\begin{equation}
  E(\bm{\alpha}, \bm{\theta}, \mathbf{z})
  = U(\bm{\alpha}, \bm{\theta}, \mathbf{z})
  + V(\bm{\alpha}, \mathbf{z}),
\end{equation}
where $\bm{\alpha} \in \{0,1\}^N$ is the binary segmentation mask,
$\bm{\theta}$ are the GMM parameters,
and $\mathbf{z}$ are the observed pixel colours.
The unary term $U$ measures how well each pixel's colour fits the foreground and background
GMMs; the pairwise term $V$ penalises label discontinuities at edges with low colour contrast.
Minimisation alternates between GMM re-estimation (EM step) and mask update via
min-cut/max-flow~\cite{boykov2001fast}.

The trimap partitions pixels into:
$\mathcal{T}_F$ (definite foreground $\to$ \texttt{GC\_FGD}),
$\mathcal{T}_B$ (definite background $\to$ \texttt{GC\_BGD}),
$\mathcal{T}_U$ (unknown $\to$ \texttt{GC\_PR\_FGD}/\texttt{GC\_PR\_BGD}).
Only pixels in $\mathcal{T}_U$ participate in the min-cut; the definite regions seed the GMMs.

\subsection{Graph Convolutional Networks}

A GCN layer~\cite{kipf2017semi} computes node representations as:
\begin{equation}
  \mathbf{H}^{(\ell+1)}
  = \sigma\!\left(\tilde{\mathbf{D}}^{-\frac{1}{2}}
                  \tilde{\mathbf{A}}
                  \tilde{\mathbf{D}}^{-\frac{1}{2}}
                  \mathbf{H}^{(\ell)}
                  \mathbf{W}^{(\ell)}\right),
\end{equation}
where $\tilde{\mathbf{A}} = \mathbf{A} + \mathbf{I}$ is the adjacency matrix with self-loops,
$\tilde{\mathbf{D}}$ the corresponding degree matrix,
$\mathbf{H}^{(\ell)}$ the node feature matrix at layer $\ell$,
and $\mathbf{W}^{(\ell)}$ a learnable weight matrix.
For interactive segmentation, this formulation naturally propagates click signals from
annotated superpixels to their spatial neighbours.

\section{Method}

\subsection{Graph Construction}

Given an input image $I \in \mathbb{R}^{H \times W \times 3}$, we apply SLIC~\cite{achanta2012slic}
to produce $N$ superpixels, each becoming a node $v_i$ in a Region Adjacency Graph
$\mathcal{G} = (\mathcal{V}, \mathcal{E})$.
Edges connect spatially adjacent superpixels (4-connectivity RAG), augmented with long-range
$k$-NN edges in colour space.

\subsubsection*{Edge features ($\mathbf{e}_{ij} \in \mathbb{R}^{4}$)}

Each edge encodes: colour dissimilarity ($\Delta E$ in LAB), normalised spatial distance
between centroids, shared boundary length (normalised), and gradient contrast at the shared
boundary.

\subsection{User Hint Encoding}

Given $n_f$ foreground clicks $\mathcal{F} = \{p_k\}$ and $n_b$ background clicks
$\mathcal{B} = \{q_k\}$, each superpixel $v_i$ receives:
\begin{equation}
  \mathbf{h}_i =
  \bigl[\mathbf{1}[v_i \cap \mathcal{F} \neq \emptyset],\;
        \mathbf{1}[v_i \cap \mathcal{B} \neq \emptyset],\;
        \mathbf{1}[\text{neither}]\bigr]^\top.
\end{equation}
This binary encoding is appended to the image features to form the full node feature
$\mathbf{x}_i \in \mathbb{R}^{19}$.

\subsection{ResGCNNet Architecture}
\label{sec:resgcnnet}

The recommended model follows the pipeline:
\[
  \mathbf{x} \;\xrightarrow{\text{InputProj}}\;
  \mathbf{h}^{(0)} \;\xrightarrow{\text{HintBooster}}\;
  \tilde{\mathbf{h}}^{(0)} \;\xrightarrow{\text{ResBlocks} \times 8}\;
  \{\mathbf{h}^{(\ell)}\} \;\xrightarrow{\text{SAGEConv}}\;
  \mathbf{h}^{(s)} \;\xrightarrow{\text{GlobalCtx}}\;
  \xrightarrow{\text{DenseConcat + Fusion}}\;
  \hat{\mathbf{y}}.
\]

Figure~\ref{fig:architecture} shows the full architecture.

\subsubsection*{Input Projection}
A linear layer followed by LayerNorm and GELU maps node features to a hidden dimension
$D = 128$:
\[
  \mathbf{h}^{(0)} = \text{GELU}(\text{LN}(\mathbf{x}\mathbf{W}_0)).
\]

\subsubsection*{HintBooster}
To amplify the signal at hint-annotated nodes before any message-passing occurs, a small
MLP produces a multiplicative gate from the hint features alone:
\[
  \mathbf{g} = \sigma\!\left(\mathbf{W}_2\,\text{ReLU}(\mathbf{W}_1\,\mathbf{h}_{-3:})\right)
  \in \mathbb{R}^D,
  \qquad
  \tilde{\mathbf{h}}^{(0)} = \mathbf{h}^{(0)} + \mathbf{g} \odot \mathbf{h}^{(0)}.
\]

\subsubsection*{Residual GCN Blocks}
Each of the $L = 8$ residual blocks computes:
\begin{align}
  \mathbf{r} &= \text{GCNConv}\!\left(\text{LN}(\mathbf{h}^{(\ell-1)}),\,
                 \mathcal{E}\right), \\
  \bm{\gamma}_{ij} &= \sigma\!\left(\text{MLP}_e(\mathbf{e}_{ij})\right),\quad
  \bm{\gamma}_i = \frac{1}{|\mathcal{N}(i)|}\sum_{j \in \mathcal{N}(i)} \bm{\gamma}_{ij},
  \label{eq:edgegate}\\
  \mathbf{h}^{(\ell)} &= \mathbf{h}^{(\ell-1)}
    + \text{Dropout}\!\left(\text{GELU}(\mathbf{r} \odot \bm{\gamma}_i)\right).
\end{align}
Equation~\eqref{eq:edgegate} is the \emph{EdgeGate}: it modulates the convolution output
with learned edge-feature weights before the residual addition, making each layer sensitive
to colour boundaries and spatial distances.

\subsubsection*{Multi-Scale Branch (SAGEConv)}
After the residual stack, a single GraphSAGE~\cite{hamilton2017inductive} layer aggregates
a coarser neighbourhood summary:
\[
  \mathbf{h}^{(s)} = \text{GELU}\!\left(\text{LN}(\text{SAGEConv}(\mathbf{h}^{(L)}, \mathcal{E}))\right).
\]

\subsubsection*{Attention-Weighted Global Context}
A graph-level summary vector is computed with an attention readout and broadcast back to
all nodes:
\[
  w_i = \frac{\exp(\mathbf{a}^\top \mathbf{h}^{(L)}_i)}
             {\sum_j \exp(\mathbf{a}^\top \mathbf{h}^{(L)}_j)},
  \quad
  \mathbf{g}_{\text{ctx}} = \sigma\!\left(\mathbf{W}_g\,
    \text{ReLU}\!\left(\mathbf{W}_c \textstyle\sum_i w_i \mathbf{h}^{(L)}_i\right)\right),
  \quad
  \mathbf{h}^{(L)} \leftarrow \mathbf{h}^{(L)} \odot \mathbf{g}_{\text{ctx}}.
\]
This gives every node awareness of the global foreground-background balance.

\subsubsection*{Dense Concatenation and Trimap Head}
All intermediate representations are concatenated:
\[
  \mathbf{z} = \bigl[\mathbf{h}^{(0)} \| \mathbf{h}^{(1)} \| \cdots \|
                      \mathbf{h}^{(L)} \| \mathbf{h}^{(s)}\bigr]
  \in \mathbb{R}^{D(L+2)},
\]
and passed through a two-layer fusion MLP followed by a linear head:
\[
  \hat{\mathbf{y}} = \mathbf{W}_{\text{head}}\,\text{Fusion}(\mathbf{z})
  \in \mathbb{R}^{3}.
\]
The three output logits correspond to classes $\{\text{BG}, \text{UNK}, \text{FG}\}$.

\subsection{Trimap Mapping}

Per-superpixel logits are converted to the four-class GrabCut label space via thresholding:
\begin{equation}
  T_i =
  \begin{cases}
    \texttt{GC\_FGD}     & \text{if } P(\text{FG}) \geq \theta_f, \\
    \texttt{GC\_BGD}     & \text{if } P(\text{BG}) \geq \theta_b, \\
    \texttt{GC\_PR\_FGD} & \text{if } P(\text{FG}) > P(\text{BG}), \\
    \texttt{GC\_PR\_BGD} & \text{otherwise},
  \end{cases}
\end{equation}
with default thresholds $\theta_f = \theta_b = 0.55$.
The predicted trimap is then upsampled to pixel resolution using the superpixel segmentation
map and passed to \texttt{cv2.grabCut}.

\section{Training}

\subsection{Dataset}

We train and evaluate on BSDS500~\cite{arbelaez2011contour}, using the standard 200-image
training split (augmented $3\times$ with random flips, rotations, colour jitter, and crops)
and 100-image validation split.
Ground-truth trimaps are derived from binary segmentation masks by assigning superpixels
with foreground overlap $\geq 0.70$ to class FG, background overlap $\geq 0.70$ to BG,
and the remainder to UNK.

\subsection{Click Simulation}

At each training iteration, $n_f \sim \mathcal{U}(3,9)$ foreground clicks and
$n_b \sim \mathcal{U}(3,9)$ background clicks are sampled from the ground-truth mask with
a small spatial jitter ($\sigma = 0.02$ of image width) and erosion radius of 6 pixels
to avoid ambiguous boundary clicks.
This stochastic click augmentation prevents the model from overfitting to fixed click patterns.

\subsection{Loss Function}

We use Focal Loss~\cite{lin2017focal} with class weights to address the imbalance between
FG/BG and the typically dominant UNK class:
\begin{equation}
  \mathcal{L} = -\sum_{i} w_{y_i}(1 - p_{y_i})^\gamma \log p_{y_i},
  \quad \gamma = 2.0,
  \quad \mathbf{w} = [1.4,\, 0.7,\, 1.4].
\end{equation}

\subsection{Optimisation}

We use AdamW~\cite{loshchilov2019decoupled} with layer-wise learning rate decay
(earlier GCN layers receive $0.8\times$ the base LR per layer from the output).
The learning rate follows a cosine annealing schedule with warm restarts ($T_0 = 50$ epochs).
Training runs for up to 120 epochs with early stopping (patience = 30) based on validation
foreground IoU.
Mixed-precision training (FP16) is used on CUDA devices.

\subsection{Evaluation}

Validation IoU is computed as the average over three independently sampled click sets
per image to reduce click-pattern variance:
\begin{equation}
  \text{IoU}_{\text{FG}} = \frac{1}{K}\sum_{k=1}^{K}
  \frac{|\hat{\mathcal{M}}_k \cap \mathcal{M}|}{|\hat{\mathcal{M}}_k \cup \mathcal{M}|},
  \quad K = 3.
\end{equation}
The best checkpoint is saved on maximum validation $\text{IoU}_{\text{FG}}$.

\section{Preliminary Results and Discussion}

Training is ongoing. Qualitative observations from early epochs suggest:

\begin{itemize}[itemsep=3pt]
  \item The \textbf{HintBooster} consistently accelerates convergence in the first 10 epochs
        compared to a baseline without it, confirming the value of amplifying hint signals
        before message-passing.
  \item \textbf{EdgeGate modulation} visibly sharpens trimap boundaries at high-contrast
        edges, reducing bleed of the FG class into background regions with similar colour.
  \item The \textbf{attention-pooled global context} appears especially helpful for images
        with small foreground objects, where a mean-pooled summary would be dominated by
        background nodes.
  \item GrabCut refinement after GCN trimap prediction consistently recovers fine boundary
        details lost at the superpixel resolution.
\end{itemize}

Quantitative results will be reported once training converges.

\section{Conclusion and Future Work}

We presented GCN-GrabCut, a hybrid framework that eliminates manual trimap annotation by
predicting it automatically from sparse user clicks via graph convolutional message-passing.
The proposed ResGCNNet integrates HintBooster conditioning, EdgeGate modulation, multi-scale
aggregation, and attention-pooled global context into a single end-to-end trainable model.

\medskip
\noindent\textbf{Future directions} include:
\begin{itemize}[itemsep=2pt]
  \item Replacing the binary hint encoding with a soft distance-based propagation
        (e.g.\ geodesic distance to the nearest click).
  \item Investigating iterative refinement: feeding the GrabCut output mask back as
        additional node features for a second GCN pass.
  \item Extending to video segmentation by adding temporal edges between frames.
  \item Benchmarking against state-of-the-art interactive segmentation methods on
        GrabCut50 and Berkeley datasets.
\end{itemize}
